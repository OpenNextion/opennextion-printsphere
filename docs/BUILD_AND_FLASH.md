# Build And Flash From Source

This document is for developers who want to build OpenNextion-printsphere from
source.

For normal users, prefer the GitHub Release full initial flashing image and see
`RELEASE_FLASHING.md`.

## Supported Source Build Target For v0.1.0

`v0.1.0` is intended for:

```text
Board:       ONX3248G035
Orientation: landscape
Target:      ESP32-S3
```

The default public source build target is also `ONX3248G035` landscape. Passing
the board and orientation explicitly is still recommended when reusing an
existing build directory, because CMake cache values can survive source changes.

## Environment

Use ESP-IDF compatible with the project dependency file. The current public
source expects ESP-IDF `>= 6.0.0`; the local release candidate was validated
with ESP-IDF `v6.0.1`.

Confirm your toolchain before building:

```sh
idf.py --version
cmake --version
ninja --version
```

## Build

From the repository root:

```sh
idf.py \
  -DPRINTSPHERE_BOARD=onx3248g035 \
  -DPRINTSPHERE_ONX_ORIENTATION=landscape \
  -DPRINTSPHERE_RELEASE_VERSION=v0.1.0 \
  build
```

To also package a full initial flashing image:

```sh
idf.py \
  -DPRINTSPHERE_BOARD=onx3248g035 \
  -DPRINTSPHERE_ONX_ORIENTATION=landscape \
  -DPRINTSPHERE_RELEASE_VERSION=v0.1.0 \
  build release_initial_flash
```

The release packaging target creates a merged full image for initial flashing.
Do not commit generated firmware files to git.

## Flash With The Standard `flash_args` Flow

Use the `flash_args` generated by the build. Do not invent alternate partition
layouts, reset modes, or write offsets.

From the build directory:

```sh
cd build
python -m esptool --chip esp32s3 \
  -p <SERIAL_PORT> \
  -b 115200 \
  --before default-reset \
  --after hard-reset \
  write-flash @flash_args
```

Replace `<SERIAL_PORT>` with the serial device for your computer, for example a
platform-specific USB serial device path.

## Serial / Flashing Rules

- Use the generated `@flash_args` from the build directory.
- Use baud `115200` unless the project release documentation explicitly changes
  it.
- Use `--before default-reset --after hard-reset`.
- Do not change partition tables, flash offsets, reset flags, or baud rate as a
  workaround for a failed flash.
- If flashing fails, record the exact command and error output, then fix the
  serial/toolchain issue before retrying.

## Release Build Consistency

For public release builds, the release version must match the intended GitHub
Release tag and firmware asset name. For example:

```text
Release tag:  v0.1.0
Asset name:   opennextion-printsphere-onx3248g035-landscape-full-v0.1.0.bin
CMake value:  PRINTSPHERE_RELEASE_VERSION=v0.1.0
```

All version fields should agree before a firmware image is published.
